How this page works (click to expand — collapsed by default)

End-to-end flow

  • Choose platform, keywords, and optional dates, then run Run crawl to fetch into the database for that source.
  • Step 1 / Step 2 only analyze data that is already ingested, using the same date filters. They do not replace crawling.

Step 1 — Lexicon hits + AI

  • Runs AI only on posts that matched the monitoring lexicon. The table lists only those rows.
  • Use this for a fast view of “keyword search + lexicon” coverage.

Step 2 — Full AI (no lexicon hit)

  • Runs full-text AI on posts with no lexicon hit, in batches (up to ~100 per run, newest first).
  • Click again for the next batch until the UI says there are no more eligible posts in range.
  • Use this to catch wording the lexicon does not cover.

Keywords & dates (crawler behaviour)

  • Keywords are stored per platform. If you remove all keywords for a platform and crawl again, the backend falls back to that platform’s default keyword list.
  • With dates set, ingestion filters by publish date. If many search results fall outside the range, the crawler keeps paging until it hits caps or the platform has no more results.
